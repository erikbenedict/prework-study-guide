HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Parent elements are what contain child elements.
  • <>section elements are semantic elements that define a section in a document that contains thematic content

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • The CSS box model as a whole applies to block boxes and defines how the different parts of a box — margin, border, padding, and content — work together to create a box that you can see on a page.
  • In CSS we have several types of boxes that generally fit into the categories of block boxes and inline boxes. The type refers to how the box behaves in terms of page flow and in relation to other boxes on the page.
  • when we create an external CSS file, we need to link it to the HTML file. ex: link rel="stylesheet" href="./assets/style.css"
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector.

Git

  • git status: checks what branch we are currently on
  • git checkout main: switches to the main branch
  • git pull origin main: pulls down the latest main branch (always a good habbit to pull the base branch into your feature branch before pushing up any code)
  • git checkout -b branch-name: creates a new branch and swithces to it
  • git add -A: adds, or "commits", the changed files in our working directory to the staging area of Git
  • git commit -m "description of changes made": commits, and associates a message with our commit (ie. "added code to HTML file")
  • git push origin name-of-branch: pushes all changes made to GitHub

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow dictates that JavaScript files are executed from top to bottom, order matters.
  • Use conditional statements and loops to control the order in which code is executed, or if it is even executed at all (if / else if).
  • An array is a single variable that is used to hold a group of data. Arrays are zero-indexed and sequential.
  • A "for" loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over. EX:
    for(var x = 0; x < shapes.length; x++) {// code block}
  • A function is simply a set of instructions that tells the computer how to perform a certain task. You must call a function by name in order to execute it. A function's name is followed by parentheses (), then we add curly brackets {} that contain the code for the task we want to perform.
